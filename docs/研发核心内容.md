### 2.2 企业质量评分模型（五维度95分制）

```
总分 = 资本实力分 + 人力规模分 + 行业分 + 区位分 + 关联分
```

| 维度 | 分值范围 | 评分标准 |
|------|---------|---------|
| **资本实力分** | 0-30分 | 注册资本：>100亿=30, >10亿=22, >1亿=14, >1000万=7, <1000万=2 |
| **人力规模分** | 0-25分 | 参保人数：>5000=25, >1000=18, >200=12, >50=6, <50=2 |
| **行业分** | 0-25分 | 金融/政府/MICE=25, 航运/能源/制造=15, IT/科研=10, 其他=5 |
| **区位分** | 0-15分 | 滨海新区=15, 天津市=8, 外省市=3 |
| **关联分** | 0-20分 | 母公司已合作=20, 招投标关联=15, 同产业园=10, 无=0 |

**分级标准**：
- P0核心攻坚：≥80分
- P1重点跟进：50-79分
- P2持续培育：30-49分
- P3观察储备：<30分

## 三、新客户开发 vs 存量客户盘活（v1.7核心创新）

### 3.1 关键区分

| 维度 | 新客户开发 | 存量客户盘活 |
|------|-----------|-------------|
| **客户类型** | 新客户（未在客户池） | 存量客户/休眠客户 |
| **销售分配** | 新分配销售 | **保持原销售不变** |
| **话术策略** | 初次拜访/建立关系 | **深化关系/增值服务/交叉销售/休眠唤醒** |
| **跟进重点** | 首次签约转化 | **复购率/客单价/休眠唤醒** |
| **工单类型** | 开发工单 | **盘活血单** |
| **跟进周期** | 4周签约节奏 | **2周盘活节奏** |
| **成功指标** | 签约率 | **复购率/客单价提升/唤醒率** |

### 3.2 存量客户盘活话术（区别于新客户）

#### 增值服务话术
> "X总您好，我是[原销售姓名]。感谢您一直以来的支持！我们酒店最近升级了深海温泉SPA服务，作为我们的VIP客户，特别邀请您免费体验一次。另外，您公司的协议价我们也在考虑升级，想了解一下您今年的会议和住宿需求有没有变化？"

#### 交叉销售话术
> "X总，注意到您公司主要使用我们的客房服务，其实我们的会议室和餐饮服务也很适合贵公司的商务活动。最近我们推出了'会议+住宿+餐饮'的打包优惠，比单独预订节省20%，要不要我给您做个方案？"

#### 休眠唤醒话术
> "X总，好久没联系了！最近我们酒店做了不少升级，特别想念您这样的老客户。我们推出了'老客户回归礼包'，包含温泉体验券和餐饮折扣，什么时候方便过来坐坐？"

#### 客单价提升话术
> "X总，您一直是我们的重要客户。我们注意到您公司的商务接待需求很高，我们新推出了VIP接待套餐（行政楼层+温泉SPA+特色宴请），比普通协议价只贵15%，但体验提升很多。要不要我给您做个对比方案？"

### 3.3 P0客户分类处理逻辑

```
P0客户列表生成后：
├── 检查客户是否已在"客户拜访记录表"中
│   ├── 已存在 → 标记为"存量客户"
│   │   ├── 查询原负责销售
│   │   ├── 生成"盘活血单"（保持原销售）
│   │   ├── 使用盘活话术（增值服务/交叉销售/休眠唤醒）
│   │   └── 2周盘活节奏
│   └── 不存在 → 标记为"新客户"
│       ├── 新分配销售（李勋/赵炳涛/陈雷明）
│       ├── 生成"开发工单"
│       ├── 使用新客户话术（初次拜访/建立关系）
│       └── 4周签约节奏
└── 检查最后拜访日期
    ├── >90天未拜访 → 标记为"休眠客户"
    │   ├── 生成"盘活血单"（保持原销售）
    │   ├── 使用休眠唤醒话术
    │   └── 2周唤醒节奏
    └── ≤90天 → 保持"存量客户"标记
```

### 6.2 4周跟进节奏检测

```
第1周（联系）：
  ├── 检测：首轮拜访日期是否已填写
  ├── 超期：首轮拜访日期为空 → 提醒销售启动
  └── 正常：更新跟进状态为"第1周-联系"

第2周（参观）：
  ├── 检测：参观日期是否已填写
  ├── 超期：参观日期为空且首轮拜访已过7天 → 提醒邀请参观
  └── 正常：更新跟进状态为"第2周-参观"

第3周（试住）：
  ├── 检测：试住日期是否已填写
  ├── 超期：试住日期为空且参观已过7天 → 提醒安排试住
  └── 正常：更新跟进状态为"第3周-试住"

第4周（签约）：
  ├── 检测：签约日期是否已填写
  ├── 超期：签约日期为空且试住已过7天 → 提醒推进签约
  └── 正常：更新跟进状态为"第4周-签约"，标记为"已完成"
```

## 十、一键部署指南（供其他酒店复用）

### 10.1 部署清单

| 步骤 | 操作 | 工具 | 预计耗时 |
|------|------|------|---------|
| 1 | 安装SKILL | `clawhub install hotel-client-dev` 或 `cp skills/hotel-client-dev ~/.qoderwork/skills/` | 1分钟 |
| 2 | 配置酒店参数 | 修改SKILL.md中的参数配置表(区域代码/酒店名称/产品包价) | 5分钟 |
| 3 | 创建飞书多维表格 | 运行初始化脚本创建客户池/工单/竞品监测表 | 2分钟 |
| 4 | 导入现有客户 | 从PMS或Excel导入现有协议客户到客户池 | 10分钟 |
| 5 | 配置飞书机器人 | 创建机器人并加入酒店销售群/客户群 | 5分钟 |

### 10.3 新开/筹开酒店快速启动

对于没有存量客户的新酒店，跳过"存量盘活"引擎，直接进入"增量拓新"：

1. 配置酒店参数(区域代码/地址/产品包价)
2. 运行爱企查全行业批量检索(一级+二级行业)
3. AI评分分级 → 生成首批P0客户开发工单
4. 配置每日拓客cron → 持续补充客户池
5. 首月目标：签约20+协议客户

### 10.4 存续经营酒店完整启动

对于有存量客户的酒店(如天津瑞湾)：

1. 配置酒店参数
2. 导入现有客户到客户池(标记"已合作")
3. 运行存量客户风险体检(首次)
4. 运行存量客户生态链深挖(每个P0客户的子公司/投资)
5. 运行爱企查增量拓客(去重后补充新客户)
6. 配置所有定时任务(竞品日报/风险体检/自动拓客/客户跟进)
7. 持续运营：双引擎驱动(存量盘活+增量拓新)
