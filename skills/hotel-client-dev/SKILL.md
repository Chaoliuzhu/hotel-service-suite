# hotel-client-dev · 酒店客户开发工单系统 v1.0

## 定位

将企查查API企业数据 + 酒店产品特色 + AI评分逻辑整合为标准化客户开发工单，实现从"凭经验找客户"到"AI算好客户+派好工单"的升级。

## 核心数据源

| 数据源 | 用途 | 接入方式 |
|--------|------|----------|
| 企查查API (qcc-company MCP) | 企业工商/高管/股东/投资/财务 | MCP工具直接调用 |
| 飞书多维表格 | 客户池/工单/台账 | lark-cli api |
| 酒店PMS | 历史消费数据 | 待接入 |
| 百达屋会员API | C端会员数据 | 待接入 |

## 三大开发维度

### 维度一：存量-增量深挖

针对已有协议客户，深挖其生态链中的关联企业：

**深挖路径（以中海油为例）：**
```
中海油能源发展股份有限公司 (已合作)
├── 二级公司: 渤海地区管理服务中心 (已合作)
├── 二级公司: 配餐服务分公司 (已合作)
├── 二级公司: 装备技术 (已合作)
├── 三级公司: 各项目部/作业区
├── 招投标中标单位: 第三方服务商
├── 合资公司: 孚宝渤化(天津)仓储 (已合作)
└── 关联供应商: 设备/材料/工程承包商
```

**企查查工具调用链：**
1. `get_company_by_query` → 锁定母公司
2. `get_external_investments` → 获取所有对外投资(子公司)
3. `get_shareholder_info` → 获取股东(合资方)
4. `get_branches` → 获取分支机构
5. `get_change_records` → 获取新增/变更项目(可能新增子公司)

**深挖评分加权：**
- 母公司已合作: +20分(信任传递)
- 招投标中标方: +15分(稳定关联)
- 同一产业园区: +10分(地理便利)

### 维度二：会务市场

专门做会务的平台和公司是高价值间接客户——他们的服务对象就是我们要开发的协议客户。

**目标企业画像：**
- 会务公司/展览公司/活动策划公司
- 与周边央国企有长期合作的会务服务商
- MICE平台(会议预订平台/会议管理SaaS)

**企查查搜索关键词：**
```
会务, 会议服务, 展览, 会展, 活动策划, MICE, 会议管理,
公关活动, 商务服务, 文化传播, 品牌策划
```

**开发逻辑：**
会务公司 → 查其服务的甲方企业 → 甲方企业成为我们的目标协议客户
会务公司本身 → 签订会议场地+住宿打包框架协议

### 维度三：增量-新拓开发

对比"现有客户池"和"潜在客户池"的差异，识别纯新客户：

```
潜在客户池(企查查全量) ─ 现有客户池(PMS+拜访记录) = 纯新客户清单
```

新客户按AI评分排序 → 分配到对应销售 → 生成新拓工单

## AI评分模型 (v1.1 — 行业×地域双维度修正)

### 核心公式
```
总分 = 规模分 + 行业分 + 区位分 + 关联分 + 行业加分
地域可达性修正系数 × 总分 = 最终得分
```

### 基础分

**规模分(0-40):**
  注册资本: >100亿=40, >10亿=25, >1亿=15, >1000万=5
  员工数: >1000人=+20, >100人=+10, >50人=+5

**行业分(0-25):**
  金融/政府/MICE: 25
  航运/能源/制造/建筑: 15
  科技/医药/教育: 10
  其他: 5

**区位分(0-15):**
  滨海新区注册: 15
  天津其他区域: 8
  外省市: 3

**关联分(0-20):**
  母公司已合作: 20
  招投标关联: 15
  同产业园: 10
  无关联: 0

### 行业加分 (v1.1新增)

针对酒店会议/住宿需求特征，按行业维度额外加分：

| 行业 | 加分 | 理由 |
|------|------|------|
| 银行/金融 | +20 | 培训密集、分行巡检、客户经理出差、年会 |
| 学校/高校 | +15 | 学术会议、招生季接待、校企合作、毕业季活动 |
| 医院/医疗 | +15 | 学术会议、医疗培训、专家来访、医药代表差旅 |
| IT/科技 | +10 | 技术沙龙、客户演示、驻场开发 |
| 新质生产力企业 | +15 | 研发经费充足、对外交流频繁、政策参观接待 |
| 央国企研究院 | +20 | 研发经费充裕、学术会议密集、政企交流频繁 |
| 会务/MICE公司 | +18 | 间接客户,签约后带来批量甲方客户 |

### 地域可达性修正 (v1.1新增)

**解决"渤海银行问题"**：总部在市区(50min车程)的大型企业，总分打折；其在滨海的分支机构单独评分。

| 实际可达性 | 修正系数 | 说明 |
|-----------|---------|------|
| 滨海新区内(≤15min车程) | ×1.0 | 不修正 |
| 滨海新区外围(15-30min) | ×0.9 | 轻微折损 |
| 天津市区(30-60min) | ×0.6 | 总部直签难度大,但滨海分支机构另算 |
| 外省市 | ×0.3 | 除非有天津分部/项目驻场 |
| 在滨海新区有分支机构 | +0.3补回 | 总部在远处但滨海有分支,按分支评分 |

**规则：评分对象是"可触达的业务单元"而非"企业集团总部"**
- 例：渤海银行总部(河东区)→ 原始98分 × 0.6 = 59分 → P2
- 但：渤海银行滨海分行 → 按分支规模重新评分,可能60-70分 → P1
- 关键：企查查搜索时要搜"渤海银行 滨海"而非"渤海银行"

### 分级标准
```
P0-核心攻坚: ≥80分 → 立即开发,领军人物(赵炳涛/陈雷明/孙雪艳)亲自跟进
P1-重点跟进: 50-79分 → 一周内启动,标准销售负责
P2-持续培育: 30-49分 → 纳入月度触达计划
P3-观察储备: <30分 → 被动响应,不做主动投入
```

## 工单7模块模板

每个P0/P1客户生成包含以下7个模块的开发工单：

### 模块1: 客户基本面
- 企业画像(企查查数据)
- 注册资本/员工/行业/区位
- 酒店商务价值评分及理由

### 模块2: 决策人信息
- 企查查高管名单(get_key_personnel)
- 对接优先级排序(行政部>办公室>分管副职>一把手)
- 建议切入路径

### 模块3: 酒店需求分析
- 6类场景(会议/接待/培训/差旅/宴请/活动)
- 频次/间夜/餐饮/特殊要求
- 核心痛点 + 差异化切入点

### 模块4: 预估间夜价值
```
大型企业(>1000人): 基础50间夜/月 × 旺季系数1.5
中型企业(100-999人): 基础20间夜/月
政府/事业单位: 基础15间夜/月 × 会议季系数2
协议房价按350元/间夜计算
餐饮附加: 间夜×0.3×人均150元
```

### 模块5: 定制话术
3个切入角度，每个包含：
- 开场白(结合企业特征)
- 需求挖掘3问
- 方案呈现(酒店产品组合)
- 异议处理预案

### 模块6: 竞品对标
- 4家核心竞品(万丽泰达/滨海一号/瑞湾开元大酒店/中心皇冠假日)
- 我方差异化优势(深海温泉SPA/杭帮菜赣菜/294间客房/五星级)
- 价格竞争力分析

### 模块7: 跟进策略
- 推荐销售(主责+协助)
- 4周跟进节奏(第1周首访→第2周方案→第3周试住→第4周签约)
- 效果追踪指标

## 酒店状态适配模板 (v1.3新增 — 可复制核心)

本SKILL支持德胧集团旗下三种酒店状态，部署时选择对应模板：

### 状态一：新开/筹开酒店（无存量客户）

```
核心目标：从0到1建立客户池
工作重心：100%增量拓新

执行流程：
1. 高德POI摸排：酒店周边5km/10km/15km所有写字楼/产业园/会展中心
2. 企查查批量筛查：按行业+注册资本+参保人数筛选
3. 行业维度扫描：银行/学校/医院/IT/研究院逐个行业攻破
4. 会务公司先行：先签会务公司，借其甲方资源快速获客
5. 竞品客户截流：查竞品酒店周边企业，主动挖角

关键指标：首月签约≥20家协议客户
```

### 状态二：存续经营酒店（有存量客户，如天津瑞湾）

```
核心目标：存量盘活 + 增量拓新 双轮驱动
工作重心：60%存量深挖 + 40%增量拓新

双引擎工作流：

引擎A - 存量盘活：
  Step1: 导出PMS/飞书台账所有现有客户名单
  Step2: 企查查批量富化(注册资本/参保人数/行业/高管/投资)
  Step3: AI重新评分(用v1.3评分模型)
  Step4: 识别"沉睡客户"(超60天无消费) → 唤醒工单
  Step5: 识别"低价值客户" → 分析是否有提升空间
  Step6: 生态链深挖 → 每个现有客户的子公司/投资方/合资方

引擎B - 增量拓新：
  Step1: 高德POI摸排酒店周边(写字楼357栋/产业园112个/会展中心28个)
  Step2: 企查查按行业搜索(银行/医院/学校/IT/研究院)
  Step3: 排除已在客户池的企业 → 纯新客户
  Step4: AI评分排序 → 生成开发工单 → 派单给销售

关键指标：存量客户复购率提升20% + 月新增协议客户≥10家
```

### 状态三：业绩低迷酒店（需快速止血）

```
核心目标：快速找到高概率签约客户
工作重心：80%存量激活 + 20%精准新拓

执行流程：
1. 优先激活"近30天询价但未成交"的客户(最近询价记录表)
2. 优先激活"曾经合作但流失"的客户(流失原因分析)
3. 优先激活"竞品合约即将到期"的客户(企查查变更监测)
4. 快速签约策略：限时优惠+试住体验+百达屋会员权益包
```

## 企业质量评分模型 (v1.3 — 注册资本+参保人数双锚定)

### 核心公式
```
总分 = 资本实力分 + 人力规模分 + 行业分 + 区位分 + 关联分
地域可达性修正系数 × 总分 = 最终得分
```

### 资本实力分(0-30) — 注册资本是"钱袋子"
| 注册资本 | 得分 | 典型企业 |
|---------|------|---------|
| >100亿元 | 30 | 渤海银行(177亿)/天津港(70亿)/中远海运天津(89亿) |
| 10-100亿 | 22 | 中海油服(48亿)/国家管网天津LNG(32亿) |
| 1-10亿 | 14 | 博迈科(2.8亿)/临港控股(54亿) |
| 1000万-1亿 | 7 | 中小型企业 |
| <1000万 | 2 | 小微企业 |

### 人力规模分(0-25) — 参保人数是"真实力"
> 参保人数比注册资本更真实：注册资本可以虚报，社保人数骗不了人

| 参保人数 | 得分 | 酒店需求 |
|---------|------|---------|
| >5000人 | 25 | 大量差旅+年度培训+年会 |
| 1000-5000 | 18 | 频繁差旅+季度培训 |
| 200-999 | 12 | 定期差旅+会议需求 |
| 50-199 | 6 | 偶尔差旅 |
| <50 | 2 | 极少差旅 |

### 行业分(0-25) — 行业决定差旅频率
| 行业类别 | 得分 | 酒店需求特征 |
|---------|------|-------------|
| 银行/金融/保险 | 25 | 培训密集/分行巡检/VIP接待 |
| 政府/事业单位 | 22 | 会议频繁/上级检查/跨区交流 |
| 央企/国企总部 | 22 | 年度大会/项目评审/接待 |
| MICE/会展/公关 | 20 | 间接客户(带来批量甲方) |
| 航运/物流/港口 | 18 | 船员住宿/客户接待/培训 |
| 能源/化工/制造 | 15 | 项目驻场/技术人员出差 |
| IT/科技/研究院 | 15 | 技术交流/客户演示/驻场开发 |
| 学校/高校 | 12 | 学术会议/招生/校企合作 |
| 医院/医疗 | 12 | 学术会议/专家来访/药企代表 |
| 其他 | 5 | 一般 |

### 区位分(0-15) — 距离决定便利性
| 距离酒店 | 得分 | 说明 |
|---------|------|------|
| ≤3km | 15 | 步行/短途可达，最佳 |
| 3-5km | 12 | 10分钟车程 |
| 5-10km | 8 | 15分钟车程 |
| 10-20km | 5 | 30分钟车程 |
| >20km | 2 | 较远 |

### 关联分(0-20) — 信任传递
- 母公司已合作: +20
- 招投标中标方: +15
- 同产业园/写字楼: +10
- 无关联: 0

### 分级标准
```
P0-核心攻坚: ≥80分 → 领军人物亲自跟进(赵炳涛/陈雷明/孙雪艳)
P1-重点跟进: 50-79分 → 标准销售负责(张惠/王鑫/夏美娟等)
P2-持续培育: 30-49分 → 月度触达+活动邀请

### 三大知识资产工作流 (v1.2核心升级)

#### 工作流一：3分钟企业画像（拜访前快速建档）

**触发**：销售说"帮我了解一下XX公司"或系统自动为新客户建档

**执行链**（6个工具并行调用，3分钟内完成）：
```
get_company_by_query("XX公司")
  → verify_company_accuracy(名称, 信用代码)    # 确认主体
  → get_company_registration_info(信用代码)     # 工商信息
  → get_company_profile(信用代码)               # 业务简介
  → get_key_personnel(信用代码)                 # 高管名单
  → get_external_investments(信用代码)          # 对外投资
  → get_branches(信用代码)                      # 分支机构
```

**输出**：结构化企业画像报告，自动写入飞书共享池「内容资源库」
**核心价值**：下次任何人再查同一家公司，直接从知识库调出，不用重做

#### 工作流二：批量筛查（展会名单/供应商池快速初筛）

**触发**：展会拿到20张名片、年度供应商招标候选名单、行业协会会员名录

**执行链**（循环调用，每家公司约15秒）：
```
for 每家企业 in 名单:
  get_company_by_query(企业名)
  get_company_registration_info(信用代码)
  AI评分(规模+行业+区位+关联)
  写入潜在客户池(评分+优先级+行业标签)

输出：按P0/P1/P2/P3分级的筛查报告
```

**酒店场景**：
- 展会后拿到20家企业名片→批量筛查→5家P1以上→生成开发工单
- 年度供应商招标→10家候选→批量尽调→排除有风险的
- 行业协会会员名录→50家→批量画像→筛选高价值目标

#### 工作流三：定期复检（已合作方月度风险扫描）

**触发**：每月1号自动执行，或手动触发

**执行链**（循环调用现有客户池所有"已合作"企业）：
```
for 每家已合作企业 in 客户池:
  get_company_registration_info(信用代码)       # 经营状态是否仍为"存续"
  get_change_records(信用代码)                  # 近30天有无重大变更
  get_actual_controller(信用代码)               # 实控人是否变更

输出：复检报告
  - 绿码：无异常，继续保持
  - 黄码：有变更但非风险（如地址变更、新增投资）
  - 红码：高风险信号（法人变更/经营异常/股权冻结）→ 预警通知
```

**酒店场景**：
- 某协议客户法人突然变更→可能管理层换血→需要重新建立关系
- 某客户新增大量对外投资→业务扩张→住宿需求可能增加
- 某客户经营状态变为"异常"→暂停信用额度→改为预付模式

## 存量客户去重逻辑 (v1.4新增)

> 核心原则：已合作客户不重复检索，转为生态链深挖模式

### 去重判断规则

```
新客户名称 → 与本地客户池逐条比对:
├── 完全匹配: 企业名称完全一致 → 标记为"已存在"
├── 包含匹配: 新名称包含已有名称(或反之) → 标记为"疑似重复"
├── 信用代码匹配: 统一社会信用代码一致 → 标记为"已存在"
├── 简称匹配: 企业简称与已有名称匹配 → 标记为"疑似重复"
└── 未匹配: 标记为"新客户" → 进入评分入库流程
```

### 存量客户处理策略

**已合作客户(存续状态)** → 不做新拓，转为以下动作：

### 4周跟进节奏检测

```
第1周（联系）：
  ├── 检测：首轮拜访日期是否已填写
  ├── 超期：首轮拜访日期为空 → 提醒销售启动
  └── 正常：更新跟进状态为"第1周-联系"

第2周（参观）：
  ├── 检测：参观日期是否已填写
  ├── 超期：参观日期为空且首轮拜访已过7天 → 提醒邀请参观
  └── 正常：更新跟进状态为"第2周-参观"

第3周（试住）：
  ├── 检测：试住日期是否已填写
  ├── 超期：试住日期为空且参观已过7天 → 提醒安排试住
