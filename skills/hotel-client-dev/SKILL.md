# hotel-client-dev · 酒店客户开发工单系统 v1.0

## 定位

将企查查API企业数据 + 酒店产品特色 + AI评分逻辑整合为标准化客户开发工单，实现从"凭经验找客户"到"AI算好客户+派好工单"的升级。

## 核心数据源

| 数据源 | 用途 | 接入方式 |
|--------|------|----------|
| 企查查API (qcc-company MCP) | 企业工商/高管/股东/投资/财务 | MCP工具直接调用 |
| 飞书多维表格 | 客户池/工单/台账 | lark-cli api |
| 酒店PMS | 历史消费数据 | 待接入 |
| 百达屋会员API | C端会员数据 | 待接入 |

## 三大开发维度

### 维度一：存量-增量深挖

针对已有协议客户，深挖其生态链中的关联企业：

**深挖路径（以中海油为例）：**
```
中海油能源发展股份有限公司 (已合作)
├── 二级公司: 渤海地区管理服务中心 (已合作)
├── 二级公司: 配餐服务分公司 (已合作)
├── 二级公司: 装备技术 (已合作)
├── 三级公司: 各项目部/作业区
├── 招投标中标单位: 第三方服务商
├── 合资公司: 孚宝渤化(天津)仓储 (已合作)
└── 关联供应商: 设备/材料/工程承包商
```

**企查查工具调用链：**
1. `get_company_by_query` → 锁定母公司
2. `get_external_investments` → 获取所有对外投资(子公司)
3. `get_shareholder_info` → 获取股东(合资方)
4. `get_branches` → 获取分支机构
5. `get_change_records` → 获取新增/变更项目(可能新增子公司)

**深挖评分加权：**
- 母公司已合作: +20分(信任传递)
- 招投标中标方: +15分(稳定关联)
- 同一产业园区: +10分(地理便利)

### 维度二：会务市场

专门做会务的平台和公司是高价值间接客户——他们的服务对象就是我们要开发的协议客户。

**目标企业画像：**
- 会务公司/展览公司/活动策划公司
- 与周边央国企有长期合作的会务服务商
- MICE平台(会议预订平台/会议管理SaaS)

**企查查搜索关键词：**
```
会务, 会议服务, 展览, 会展, 活动策划, MICE, 会议管理,
公关活动, 商务服务, 文化传播, 品牌策划
```

**开发逻辑：**
会务公司 → 查其服务的甲方企业 → 甲方企业成为我们的目标协议客户
会务公司本身 → 签订会议场地+住宿打包框架协议

### 维度三：增量-新拓开发

对比"现有客户池"和"潜在客户池"的差异，识别纯新客户：

```
潜在客户池(企查查全量) ─ 现有客户池(PMS+拜访记录) = 纯新客户清单
```

新客户按AI评分排序 → 分配到对应销售 → 生成新拓工单

## AI评分模型 (v1.1 — 行业×地域双维度修正)

### 核心公式
```
总分 = 规模分 + 行业分 + 区位分 + 关联分 + 行业加分
地域可达性修正系数 × 总分 = 最终得分
```

### 基础分

**规模分(0-40):**
  注册资本: >100亿=40, >10亿=25, >1亿=15, >1000万=5
  员工数: >1000人=+20, >100人=+10, >50人=+5

**行业分(0-25):**
  金融/政府/MICE: 25
  航运/能源/制造/建筑: 15
  科技/医药/教育: 10
  其他: 5

**区位分(0-15):**
  滨海新区注册: 15
  天津其他区域: 8
  外省市: 3

**关联分(0-20):**
  母公司已合作: 20
  招投标关联: 15
  同产业园: 10
  无关联: 0

### 行业加分 (v1.1新增)

针对酒店会议/住宿需求特征，按行业维度额外加分：

| 行业 | 加分 | 理由 |
|------|------|------|
| 银行/金融 | +20 | 培训密集、分行巡检、客户经理出差、年会 |
| 学校/高校 | +15 | 学术会议、招生季接待、校企合作、毕业季活动 |
| 医院/医疗 | +15 | 学术会议、医疗培训、专家来访、医药代表差旅 |
| IT/科技 | +10 | 技术沙龙、客户演示、驻场开发 |
| 新质生产力企业 | +15 | 研发经费充足、对外交流频繁、政策参观接待 |
| 央国企研究院 | +20 | 研发经费充裕、学术会议密集、政企交流频繁 |
| 会务/MICE公司 | +18 | 间接客户,签约后带来批量甲方客户 |

### 地域可达性修正 (v1.1新增)

**解决"渤海银行问题"**：总部在市区(50min车程)的大型企业，总分打折；其在滨海的分支机构单独评分。

| 实际可达性 | 修正系数 | 说明 |
|-----------|---------|------|
| 滨海新区内(≤15min车程) | ×1.0 | 不修正 |
| 滨海新区外围(15-30min) | ×0.9 | 轻微折损 |
| 天津市区(30-60min) | ×0.6 | 总部直签难度大,但滨海分支机构另算 |
| 外省市 | ×0.3 | 除非有天津分部/项目驻场 |
| 在滨海新区有分支机构 | +0.3补回 | 总部在远处但滨海有分支,按分支评分 |

**规则：评分对象是"可触达的业务单元"而非"企业集团总部"**
- 例：渤海银行总部(河东区)→ 原始98分 × 0.6 = 59分 → P2
- 但：渤海银行滨海分行 → 按分支规模重新评分,可能60-70分 → P1
- 关键：企查查搜索时要搜"渤海银行 滨海"而非"渤海银行"

### 分级标准
```
P0-核心攻坚: ≥80分 → 立即开发,领军人物(赵炳涛/陈雷明/孙雪艳)亲自跟进
P1-重点跟进: 50-79分 → 一周内启动,标准销售负责
P2-持续培育: 30-49分 → 纳入月度触达计划
P3-观察储备: <30分 → 被动响应,不做主动投入
```

- **酒店需求特征**: 学术会议、医疗培训、专家来访、医药代表差旅、年会

#### 4. IT/科技

#### 5. 新质生产力企业

#### 6. 央国企研究院
- **搜索策略**: "天津 研究院/实验室/创新中心/技术中心/研发中心" + 央国企关键词
- **酒店需求特征**: 学术会议密集、政企交流频繁、科研评审接待、成果发布
- **话术方向**: "研究院的学术交流和政企接待,我们提供从会场到住宿的全流程服务"

#### 7. 会务/MICE公司
- **搜索策略**: "天津 会务/会议服务/展览/活动策划/公关活动"

## 工单7模块模板

每个P0/P1客户生成包含以下7个模块的开发工单：

### 模块1: 客户基本面
- 企业画像(企查查数据)
- 注册资本/员工/行业/区位
- 酒店商务价值评分及理由

### 模块2: 决策人信息
- 企查查高管名单(get_key_personnel)
- 对接优先级排序(行政部>办公室>分管副职>一把手)
- 建议切入路径

### 模块3: 酒店需求分析
- 6类场景(会议/接待/培训/差旅/宴请/活动)
- 频次/间夜/餐饮/特殊要求
- 核心痛点 + 差异化切入点

### 模块4: 预估间夜价值
```
大型企业(>1000人): 基础50间夜/月 × 旺季系数1.5
中型企业(100-999人): 基础20间夜/月
政府/事业单位: 基础15间夜/月 × 会议季系数2
协议房价按350元/间夜计算
餐饮附加: 间夜×0.3×人均150元
```

### 模块5: 定制话术
3个切入角度，每个包含：
- 开场白(结合企业特征)
- 需求挖掘3问
- 方案呈现(酒店产品组合)
- 异议处理预案

### 模块6: 竞品对标
- 4家核心竞品(万丽泰达/滨海一号/瑞湾开元大酒店/中心皇冠假日)
- 我方差异化优势(深海温泉SPA/杭帮菜赣菜/294间客房/五星级)
- 价格竞争力分析

### 模块7: 跟进策略
- 推荐销售(主责+协助)
- 4周跟进节奏(第1周首访→第2周方案→第3周试住→第4周签约)
- 效果追踪指标

## 百达屋会员营销嵌入

每个工单必须包含百达屋会员引导话术：

**B端(企业签约时)：**
"签约协议价的同时，我们可以帮贵司员工批量开通百达屋会员——积分可以兑换我们深海温泉SPA、餐饮券、客房升级。您的员工出差住我们酒店的同时还能享受会员权益，这是其他酒店没有的。"

**C端(个人入住时)：**
"您今天入住就可以扫码开通百达屋会员，今晚的深海温泉SPA直接享会员价8折，下次预订客房还能积分抵扣。"

**话术核心原则：**
- 不是推销会员卡，而是"解锁入住增值体验"
- 强调即时权益(温泉8折/餐饮券/升房)而非远期积分
- B端强调"员工福利零成本"，C端强调"当场省钱"
